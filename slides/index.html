01 · gancho

Quantas vezes um agente
parou no meio da tarefa?

Ou "esqueceu" o que já tinha feito assim que você chamou de novo. Hoje vamos ver por quê — e três formas de resolver isso, em camadas.

o cenário

Um Analista de Vendas com dados de receita mensal. O mesmo agente de negócio nas 3 demos — só a camada em volta dele muda.

a pergunta que guia tudo

Todo agente que faz algo de verdade é, por baixo, um loop. O quanto desse loop você controla é o que muda entre as camadas.

03 · demo 1 — agent simples

Modelo + tools. Só isso.

Program.cs
AIAgent agent = chatClient
    .AsAIAgent(
        name: "AnalistaDeVendas",
        instructions: "...",
        tools:
        [
            AIFunctionFactory.Create(SalesTools.GetRevenue),
            AIFunctionFactory.Create(SalesTools.GetTopProducts),
        ]);

// funciona bem — resposta em uma tacada
await agent.RunAsync("Receita de outubro?");

// pede plano + arquivo. o texto sai, mas...
await agent.RunAsync("Compare, planeje e salve em .md");

funciona

Tool-calling: chamar a tool, devolver o resultado, responder. Perfeito para pergunta única.

não existe

Plano visível, arquivo em disco, ou memória entre execuções — cada RunAsync começa do zero.

03 · demo 2 — agent + harness

Mesmo agente. Motor batteries-included.

Program.cs
AIAgent agent = chatClient.AsHarnessAgent(
    new HarnessAgentOptions
    {
        ChatOptions = /* mesmas tools */,

        // pasta real no disco, nao sandbox
        FileAccessStore =
            new FileSystemAgentFileStore(outputDir),
    });

// escreve relatorio.md de verdade
await agent.RunStreamingAsync(input, session);

o momento "uau"

O mesmo pedido da demo 1 — mas agora relatorio.md aparece no disco, escrito pelo próprio agente, com plano e todo-list visíveis no stream.

03 · demo 3 — loop engineering

Você decide o que significa "terminado".

Program.cs
string[] criteria =
[
    "Tabela markdown comparando os meses",
    "Secao 'Riscos' com 2+ itens",
    "Recomendacao de acao clara",
];

var judge = new AIJudgeLoopEvaluator(
    judgeClient, new() { Criteria = criteria });

AIAgent loopedAgent = new LoopAgent(
    workerAgent, judge,
    new LoopAgentOptions { MaxIterations = 3 });
rascunho → juiz avalia critérios → reprovado? feedback → nova tentativa

para quando os critérios passam — ou em 3 iterações, o teto de segurança

04 · fechamento

Quando usar qual camada

CamadaUse quando...
Agent simplespergunta única, resposta em uma tomada.
Harnesstarefa multi-passo, precisa de plano / memória / arquivo — mas você confia no bom senso do modelo pra saber quando parou.
Loop engineeringexiste um critério de qualidade explícito e você não pode confiar só no modelo pra decidir "terminei".
Workflowso caminho de execução precisa ser determinístico — vários agentes, ordem fixa, checkpoint.
